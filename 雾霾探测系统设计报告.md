## 一、任务与需求分析

### 1.1 任务背景
雾霾天气会影响出行与健康。本系统的目标是在出行前提供“定位城市 + 天气详情 + 空气质量指数（AQI）”的一体化查询能力，并在移动端友好展示。

### 1.2 需求分解（逐条对应题目）
1. **定位功能**：获取定位城市；定位城市需**保存在服务器端**；并在客户端实时显示。
2. **界面设计**：移动端适配；动态展示天气与空气质量指数。
3. **天气详情与 AQI**：以定位城市作为参数，服务器端获取天气与 AQI 数据后返回客户端，并将结果**保存在服务器端**。
4. **完成报告**：给出方案设计与论证、结果与分析、测试记录与评分表。

> 说明对齐：题目允许使用百度地图/高德地图辅助定位，天气可使用和风天气、墨迹天气、我的天气等。当前实现选用：**百度地图 Web JS API 定位** + **和风天气（QWeather）天气/空气质量接口**。

## 二、总体方案设计与论证（20 分）

### 2.1 系统架构
采用轻量 B/S 架构（移动端 Web App）：

- **客户端（HTML5 Web App）**：负责定位、展示、图表渲染与交互。
- **服务器端（Flask）**：提供 REST API；负责持久化定位与天气/AQI 数据；对接第三方天气平台 API。

该结构能满足“手机像素适配（HTML5）”与“数据保存在服务器端（Flask 落库）”两项硬要求，且部署方式灵活（局域网访问或云端部署）。

### 2.2 关键模块设计

#### 2.2.1 定位模块（百度地图 API）
- 采用 `BMap.LocalCity()` 获取当前网络环境下的城市名（如“南京市”“北京市”）。
- 客户端将城市名通过 `POST /api/location` 上传至服务器，服务器写入 `server_data.json`。
- 客户端 Header 位置区域同步显示定位城市。

#### 2.2.2 天气与 AQI 模块（和风天气 QWeather）
服务器端在 `GET /api/weather?city=城市名` 被调用时：
1. 调用 QWeather **Geo 城市查找接口**获取 Location ID。
2. 调用 QWeather **实时天气接口**获取温度/天气文本/湿度。
3. 调用 QWeather **空气质量接口**获取 AQI。
4. 调用 QWeather **24 小时预报接口**取未来 5 个时间点，形成折线图数据。

随后服务器将组装后的统一 JSON 结构保存进 `server_data.json`（`weather` 字段），并返回给客户端进行 UI 渲染。

#### 2.2.3 数据持久化模块（服务器端保存）
服务器使用 JSON 文件作为轻量“数据库”，文件名为 `server_data.json`。该方式可直接证明“定位城市/天气/AQI 均保存在服务器端”。

数据模型示例：

```json
{
    "city": "南京市",
    "weather": {
        "city": "南京市",
        "temperature": "26",
        "condition": "多云",
        "humidity": "65",
        "aqi": 78,
        "forecast": [
            {"time": "08:00", "temp": 24, "humidity": 70}
        ]
    }
}
```

### 2.3 接口设计（REST API）

#### 2.3.1 保存定位城市
- **URL**：`POST /api/location`
- **请求体**：`{"city": "南京市"}`
- **响应**：`{"status": "success", "city": "南京市"}`
- **服务器行为**：写入 `server_data.json` 的 `city` 字段。

#### 2.3.2 获取天气与 AQI
- **URL**：`GET /api/weather?city=南京市`
- **响应字段**：`temperature`、`condition`、`humidity`、`aqi`、`forecast`（折线图数据）。
- **服务器行为**：对接 QWeather 获取数据，写入 `server_data.json` 的 `weather` 字段。

## 三、界面设计与实现说明（80 分中的 UI 与功能实现）

### 3.1 页面结构（HTML5 自适应）
客户端页面由 Header 与 Body 两部分组成：

- **Header**：定位图标 + 当前城市 + “刷新”按钮（重新定位 + 重新请求天气/AQI）。
- **Body**：
    - 实时温度、天气文字与动态图标。
    - AQI 数值与等级徽章（按区间映射“优/良/轻度污染/中度污染/重度污染/严重雾霾”）。
    - 湿度显示。
    - ECharts 折线图：温度曲线 + 湿度曲线（带渐变面积）。

页面使用 HTML5 视口 `viewport-fit=cover` 与响应式布局（Flex/Grid），在不同分辨率手机上保持一致。

### 3.2 动态展示与交互
- 页面加载时自动执行定位与数据请求，并显示全屏 Loading 动画。
- 获取到数据后，动态更新温度、湿度、天气描述、AQI 等信息。
- 当 AQI > 150 时，页面背景切换为灰暗色调以强化预警体验。

## 四、运行步骤与关键配置

### 4.1 环境与依赖
依赖位于 `requirements.txt`，核心依赖：Flask、Requests。

### 4.2 百度地图 AK 与 Referer 白名单
1. 在百度地图开放平台创建 Web 应用并获取 AK。
2. 将 AK 填入前端页面引入脚本：
     `https://api.map.baidu.com/api?v=3.0&ak=你的AK`
3. Referer 白名单建议至少包含：
     - `localhost`
     - `127.0.0.1`
     - 局域网 IP（示例：`10.198.204.54`）
     - 若部署到域名：`your-domain.com`、`*.your-domain.com`

### 4.3 和风天气 QWeather KEY
在服务器端配置 `QWEATHER_API_KEY`。KEY 为个人申请的开发者密钥，用于调用 QWeather 的 Geo/天气/AQI 接口。

### 4.4 本地运行
在项目根目录运行：

```powershell
\.venv\Scripts\python.exe app.py
```

浏览器访问：`http://127.0.0.1:5000`。

手机访问（同一 Wi-Fi）：访问 `http://<电脑局域网IP>:5000`。

> 若需实验室“空白电脑直接访问”，建议将本项目部署至云服务器或 PaaS 平台（浏览器直接访问域名/公网 IP），无需安装任何内网穿透客户端。

## 五、结果与分析（5 分）

### 5.1 运行结果
1. 定位完成后，Header 显示定位城市；服务器端 `server_data.json` 的 `city` 字段更新。
2. 天气与 AQI 请求成功后，页面展示温度、湿度、天气情况、AQI 等级；`server_data.json` 的 `weather` 字段写入完整查询结果。
3. 折线图显示未来时间段温度/湿度变化趋势，满足“动态显示 + 图表”要求。

### 5.2 分析
系统将“定位 -> 服务器保存 -> 服务器取天气/AQI -> 服务器保存 -> 客户端展示”形成闭环，覆盖题目四项实验要求。由于采用 HTML5 响应式布局，能在不同手机像素/分辨率下稳定展示。
